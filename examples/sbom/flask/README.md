# `flask` SBOM example

SPDX tag-value ([`sbom.spdx`](sbom.spdx)) and CycloneDX JSON ([`sbom.cdx.json`](sbom.cdx.json)) software bills of materials produced by [Provenant](https://github.com/getprovenant/provenant) for [flask](https://github.com/pallets/flask) (Python / PyPI).

## Provenance

- Source: `https://github.com/pallets/flask.git` at tag `3.1.3`
- Pinned commit: `22d924701a6ae2e4cd01e9a15bbaf3946094af65`
- Generated by: Provenant 1.0.1
- Command: `provenant scan <checkout> --license --package --copyright --strip-root --spdx-tv sbom.spdx --cyclonedx sbom.cdx.json`

## What's inside

- **54 components** — the project plus every resolved dependency, each with a package URL (purl).
- **54 dependency relationships**, fully resolved: every `dependsOn` edge points to a component in this same document (no dangling references). A packages-only SBOM lists just the top-level package(s).

## How it compares

Verified against ScanCode with the `compare-outputs` xtask; no medium-or-major license, copyright, or package regression. Notable source-faithful differences:

- Complete inventory: every resolved dependency is promoted to a component with a purl and the dependency graph is closed, so the SBOM never references materials it does not also list.
- Cleaner declared license: Provenant resolves `BSD-3-Clause`, where ScanCode reports `BSD-3-Clause AND LicenseRef-scancode-unknown-license-reference`.
- Precise package identity: the package carries a proper `pkg:pypi/flask@3.1.3` purl.
